Tại Sao Code 1 Hiệu Quả Hơn?
Dựa trên các khác biệt trên, Code 1 đạt được kết quả phân cụm hiệu quả hơn Code 2 vì các lý do sau:

Dữ Liệu Sạch Hơn:
Code 1 loại bỏ outliers bằng phương pháp IQR, đảm bảo dữ liệu đầu vào chất lượng cao hơn. Code 2 không xử lý outliers, dẫn đến khả năng các giá trị bất thường làm nhiễu kết quả phân cụm.
Tập Trung Vào Biến Mục Tiêu Quan Trọng:
Code 1 sử dụng trực tiếp G3 (điểm cuối kỳ) làm biến mục tiêu, phù hợp hơn với mục tiêu đánh giá hiệu suất học sinh. Code 2 sử dụng G_mean, có thể làm mất thông tin cụ thể về G3 và gây trùng lặp thông tin với G1 và G2.
Tiền Xử Lý Dữ Liệu Hiệu Quả:
Code 1 sử dụng ColumnTransformer để xử lý đồng nhất các cột số và định tính, giảm nguy cơ lỗi. Code 2 xử lý thủ công hơn, dễ xảy ra sai sót trong việc quản lý các cột.
Trực Quan Hóa và Phân Tích Sâu Sắc:
Code 1 cung cấp trực quan hóa PCA 2D/3D, biểu đồ boxplot, và phân tích trọng số PCA, giúp hiểu rõ hơn về các đặc trưng quan trọng và sự phân tách của các cụm. Code 2 thiếu trực quan hóa 3D và các biểu đồ chi tiết như boxplot, làm giảm khả năng diễn giải.
Giao Diện và Trình Bày Kết Quả:
Code 1 có giao diện Streamlit đơn giản, tập trung, và dễ sử dụng. Code 2 phức tạp hơn với nhiều tab và phân tích bổ sung, nhưng không tận dụng triệt để, gây khó khăn cho người dùng.
Ổn Định của Thuật Toán KMeans:
Code 1 sử dụng các tham số cố định (n_init=10, random_state=42) để đảm bảo kết quả KMeans ổn định. Code 2 sử dụng n_init='auto', có thể dẫn đến sự không nhất quán trong kết quả.
